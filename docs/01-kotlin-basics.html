1 변수: val / var

Python
# Python: 모든 변수는 재할당 가능
name = "Bitcoin"
name = "Ethereum"   # OK

price = 50000
price = 60000       # OK

# "상수"는 관례적으로 대문자
MAX_LEVERAGE = 100  # 바꿀 수 있지만 안 바꾸기로 약속
Kotlin
// val = 재할당 불가 (읽기 전용)
val name = "Bitcoin"
// name = "Ethereum"  // 컴파일 에러!

// var = 재할당 가능
var price = 50000
price = 60000        // OK

// 상수 (컴파일 타임 상수)
const val MAX_LEVERAGE = 100
핵심 요약

val = 한 번 할당하면 끝 (Python에서 "이 변수 바꾸지 마세요"를 컴파일러가 강제).

기본적으로 val을 쓰고, 꼭 바꿔야 할 때만 var를 씁니다.

2 타입 시스템: 정적 타입 vs 동적 타입

Python (동적 타입)
# 타입을 안 써도 됨
price = 50000
name = "BTC"

# 실행 시점에 타입 에러 발견
price = "오만원"  # OK... 실행은 됨
result = price + 100  # TypeError! (런타임)

# 타입 힌트 (선택사항, 강제 아님)
price: int = 50000
name: str = "BTC"
Kotlin (정적 타입 + 타입 추론)
// 타입 추론: 컴파일러가 알아서 파악
val price = 50000    // Int로 추론
val name = "BTC"     // String으로 추론

// 타입 불일치 → 컴파일 에러!
// price = "오만원"  // 에러: String ← Int

// 명시적 타입 선언도 가능
val price2: Int = 50000
val name2: String = "BTC"
val ratio: Double = 0.05

Tip: 왜 정적 타입이 좋은가

Python에서 mypy를 쓰면 타입 검사를 할 수 있지만, 선택사항이고 런타임에 강제되지 않습니다. Kotlin은 컴파일 시점에 타입 에러를 100% 잡아냅니다. "이 함수에 str을 넣어야 하는데 int를 넣었네?" 같은 버그가 코드를 실행하기도 전에 발견됩니다.

3 Null Safety

Python에서 가장 흔한 에러 중 하나: AttributeError: 'NoneType' has no attribute 'xxx'. Kotlin은 이 문제를 타입 시스템 수준에서 원천 차단합니다.

Python: None은 어디든 들어감
# None이 들어올 수 있는지 코드만 봐서는 모름
def find_user(id: int) -> User | None:
    ...

user = find_user(42)
# user가 None일 수도 있는데...
print(user.name)  # AttributeError! (런타임)
Kotlin: null 가능 여부가 타입에 명시
// String  → null 불가
// String? → null 가능 (물음표!)
val name: String = "BTC"
// name = null  // 컴파일 에러!

val nickname: String? = null  // OK

// null일 수 있는 값은 바로 사용 불가
// nickname.length  // 컴파일 에러!

Null 처리 연산자들

연산자 이름 의미 Python 대응
?. Safe call null이면 전체가 null, 아니면 실행 x.y if x else None
?: Elvis 연산자 null이면 오른쪽 기본값 사용 x if x is not None else default
!! Non-null assertion "null 아님을 보장" (null이면 예외 발생) assert x is not None; x
?.let { } Safe call + 스코프 null이 아닐 때만 블록 실행 if x is not None: ...
Python
# Safe navigation (수동)
name = user.name if user else None
length = len(name) if name else 0

# 기본값
display_name = name if name else "익명"

# None이 아님을 단언
assert user is not None
print(user.name)

# None이 아닐 때만 처리
if user is not None:
    print(user.name)
Kotlin
// ?. (Safe call)
val name = user?.name       // user가 null이면 name도 null
val length = name?.length ?: 0  // 체이닝 가능

// ?: (Elvis 연산자)
val displayName = name ?: "익명"

// !! (Non-null assertion, 위험!)
val userName = user!!.name  // null이면 NPE 발생

// ?.let { } (null이 아닐 때만)
user?.let { u ->
    println(u.name)
}
주의: !!는 최후의 수단

!!는 "null이 아님을 내가 보장한다"는 의미입니다. 실제로 null이면 NullPointerException이 발생합니다. Python의 assert처럼 방어적이지 않으므로, 가능하면 ?.이나 ?:를 사용하세요.

핵심 요약

Kotlin에서는 null이 될 수 있는 타입(?)과 없는 타입이 다릅니다.
String vs String? = 별개의 타입.
덕분에 NoneType has no attribute 류의 런타임 에러가 컴파일 시점에 잡힙니다.

4 함수

Python
# 기본 함수
def greet(name: str) -> str:
    return f"Hello, {name}!"

# 기본값 매개변수
def create_order(
    symbol: str,
    size: int = 1,
    leverage: int = 5
) -> dict:
    return {
        "symbol": symbol,
        "size": size,
        "leverage": leverage
    }

# Named argument
order = create_order(
    symbol="BTC_USDT",

)

# 한 줄 함수 (드물게 사용)
double = lambda x: x * 2
Kotlin
// 기본 함수 (fun 키워드!)
fun greet(name: String): String {
    return "Hello, $name!"
}

// 기본값 매개변수 (Python과 동일!)
fun createOrder(
    symbol: String,
    size: Int = 1,
    leverage: Int = 5
): Map<String, Any> {
    return mapOf(
        "symbol" to symbol,
        "size" to size,
        "leverage" to leverage
    )
}

// Named argument (Python과 동일!)
val order = createOrder(
    symbol = "BTC_USDT",
    leverage = 10
)

// 한 줄 함수 (= 으로 바로 반환)
fun double(x: Int) = x * 2

문자열 템플릿

Python: f-string
name = "BTC"
price = 50000
print(f"{name} 가격: ${price}")
print(f"2배: ${price * 2}")
Kotlin: 문자열 템플릿
val name = "BTC"
val price = 50000
println("$name 가격: $price")
println("2배: ${price * 2}")  // 수식은 ${}
Tip

Kotlin의 함수 문법은 Python과 상당히 비슷합니다. 기본값, named argument 모두 지원합니다. 가장 큰 차이점: 반환 타입을 명시해야 하고, fun 키워드를 사용합니다.

5 data class vs @dataclass

Python
from dataclasses import dataclass

@dataclass
class Order:
    symbol: str
    size: int
    leverage: int = 5

order1 = Order("BTC_USDT", 1)
order2 = Order("BTC_USDT", 1)

# __eq__ 자동 생성
print(order1 == order2)  # True

# __repr__ 자동 생성
print(order1)
# Order(symbol='BTC_USDT', size=1, leverage=5)

# copy는 직접 만들어야 함
from dataclasses import replace
order3 = replace(order1, leverage=10)
Kotlin
data class Order(
    val symbol: String,
    val size: Int,
    val leverage: Int = 5
)

val order1 = Order("BTC_USDT", 1)
val order2 = Order("BTC_USDT", 1)

// equals() 자동 생성
println(order1 == order2)  // true

// toString() 자동 생성
println(order1)
// Order(symbol=BTC_USDT, size=1, leverage=5)

// copy() 자동 생성 (매우 편리!)
val order3 = order1.copy(leverage = 10)
핵심 요약

Kotlin data class = Python @dataclass와 거의 동일.
자동 생성: equals(), hashCode(), toString(), copy()
copy()는 특히 편리합니다 — 불변 객체의 일부 필드만 바꾼 복사본을 쉽게 만듭니다.

6 when 표현식

Python: if-elif-else / match
# if-elif-else
def get_risk(leverage: int) -> str:
    if leverage <= 3:
        return "낮음"
    elif leverage <= 10:
        return "중간"
    elif leverage <= 50:
        return "높음"
    else:
        return "매우 높음"

# Python 3.10+ match (구조적 패턴 매칭)
def describe_status(code: int) -> str:
    match code:
        case 200:
            return "성공"
        case 404:
            return "없음"
        case _:
            return "알 수 없음"
Kotlin: when (더 강력한 switch)
// when은 값을 반환하는 "표현식"
fun getRisk(leverage: Int): String =
    when {
        leverage <= 3  -> "낮음"
        leverage <= 10 -> "중간"
        leverage <= 50 -> "높음"
        else           -> "매우 높음"
    }

// 값 매칭
fun describeStatus(code: Int): String =
    when (code) {
        200      -> "성공"
        404      -> "없음"
        in 500..599 -> "서버 에러"  // 범위!
        else     -> "알 수 없음"
    }

// 타입 매칭도 가능!
fun describe(x: Any): String =
    when (x) {
        is String -> "문자열: $x"
        is Int    -> "숫자: $x"
        else      -> "뭔지 모름"
    }
Tip

when은 Python의 match보다 더 유연합니다. 범위(in 1..10), 타입(is String), 조건식 모두 사용 가능하고, 값을 반환하는 표현식이라 변수에 바로 할당할 수 있습니다.

7 sealed class vs Python Enum

sealed class는 "이 클래스를 상속할 수 있는 것들이 정해져 있다"는 선언입니다.

Python: Enum (값만 가능)
from enum import Enum

class OrderStatus(Enum):
    PENDING = "pending"
    FILLED = "filled"
    CANCELLED = "cancelled"

# 각 상태에 추가 데이터를 넣기 어려움
# FILLED에 체결 가격을 넣고 싶으면?
# → 별도 클래스를 만들어야 함

status = OrderStatus.FILLED
if status == OrderStatus.FILLED:
    print("체결됨")
elif status == OrderStatus.CANCELLED:
    print("취소됨")
Kotlin: sealed class (데이터 포함 가능!)
sealed class OrderStatus {
    data object Pending : OrderStatus()
    data class Filled(
        val price: Double,     // 체결 가격
        val filledAt: Long     // 체결 시각
    ) : OrderStatus()
    data class Cancelled(
        val reason: String     // 취소 사유
    ) : OrderStatus()
}

// when과 조합하면 강력!
fun describe(status: OrderStatus): String =
    when (status) {
        is OrderStatus.Pending ->
            "대기 중"
        is OrderStatus.Filled ->
            "${status.price}에 체결"
        is OrderStatus.Cancelled ->
            "취소: ${status.reason}"
        // else 불필요! 컴파일러가 모든 케이스 확인
    }
핵심 요약

sealed class = Enum + 각 항목이 서로 다른 데이터를 가질 수 있음.
when과 함께 쓰면 모든 케이스를 빠짐없이 처리하는지 컴파일러가 검사합니다.
Python의 Enum은 단순한 값 나열에 그치지만, sealed class는 상태 + 데이터를 함께 표현합니다.

8 컬렉션

Python
prices = [50000, 48000, 52000, 47000]

# filter
high = [p for p in prices if p > 49000]
# [50000, 52000]

# map
doubled = [p * 2 for p in prices]
# [100000, 96000, 104000, 94000]

# sum
total = sum(prices)

# sort
sorted_prices = sorted(prices)

# dict groupby
from itertools import groupby
contracts = ["BTC_USDT", "ETH_USDT", "BTC_KRW"]
by_base = {}
for c in contracts:
    base = c.split("_")[0]
    by_base.setdefault(base, []).append(c)
# {'BTC': ['BTC_USDT', 'BTC_KRW'], 'ETH': ['ETH_USDT']}
Kotlin
val prices = listOf(50000, 48000, 52000, 47000)

// filter
val high = prices.filter { it > 49000 }
// [50000, 52000]

// map
val doubled = prices.map { it * 2 }
// [100000, 96000, 104000, 94000]

// sum
val total = prices.sum()

// sort
val sortedPrices = prices.sorted()

// groupBy (한 줄로!)
val contracts = listOf("BTC_USDT", "ETH_USDT", "BTC_KRW")
val byBase = contracts.groupBy {
    it.split("_")[0]
}
// {BTC=[BTC_USDT, BTC_KRW], ETH=[ETH_USDT]}

자주 쓰는 컬렉션 함수 대응표

작업 Python Kotlin
필터[x for x in list if ...]list.filter { ... }
변환[f(x) for x in list]list.map { ... }
필터+변환두 번 리스트 컴프리헨션list.filter { }.map { }
합계sum(list)list.sum()
최대/최소max(list)list.max()
존재 확인any(x for x in list if ...)list.any { ... }
전부 확인all(x for x in list if ...)list.all { ... }
그룹핑itertools.groupby (복잡)list.groupBy { ... }
첫 번째next(x for x in list if ...)list.first { ... }

Tip: it이란?

람다에 매개변수가 하나뿐이면 it이라는 암시적 이름을 사용합니다. prices.filter { it > 49000 }에서 it은 각 가격 값입니다. Python의 리스트 컴프리헨션에서 for x in list의 x와 같은 역할입니다.

9 확장 함수

Python에는 없는 개념! 기존 클래스를 수정하지 않고 새 메서드를 추가할 수 있습니다.

Python: 기존 클래스에 메서드 추가 (꼼수)
# Python에서는 이런 게 불가능하거나 꼼수
# str에 메서드를 추가하고 싶다면?

# 방법 1: 별도 유틸 함수 (깔끔하지 않음)
def to_contract_pair(s: str) -> tuple:
    base, quote = s.split("_")
    return (base, quote)

pair = to_contract_pair("BTC_USDT")

# 방법 2: 상속 (과도한 방법)
class ContractStr(str):
    def to_pair(self):
        return self.split("_")
Kotlin: 확장 함수 (깔끔!)
// String에 새 메서드 추가!
fun String.toContractPair(): Pair<String, String> {
    val (base, quote) = this.split("_")
    return Pair(base, quote)
}

// 마치 원래 있던 메서드처럼 사용
val pair = "BTC_USDT".toContractPair()
// Pair("BTC", "USDT")

// 실용 예: Double에 퍼센트 변환 추가
fun Double.toPercent(): String =
    "%.2f%%".format(this * 100)

val ratio = 0.053
println(ratio.toPercent())  // "5.30%"
핵심 요약

확장 함수 = 기존 클래스의 소스 코드를 건드리지 않고 메서드를 추가하는 기능.
fun 타입.함수명() 형태로 선언. 유틸리티 함수를 객체의 메서드처럼 쓸 수 있어서 코드가 읽기 좋아집니다.

10 람다 / 고차 함수

Python
# 람다 (한 줄만 가능)
double = lambda x: x * 2

# 고차 함수
def apply(value, func):
    return func(value)

result = apply(5, lambda x: x * 2)  # 10

# map/filter에 람다
prices = [50000, 48000, 52000]
high = list(filter(
    lambda p: p > 49000, prices
))

# 여러 줄이면 일반 함수
def process(price):
    tax = price * 0.1
    fee = price * 0.01
    return price - tax - fee

results = list(map(process, prices))
Kotlin
// 람다 (여러 줄 가능!)
val double = { x: Int -> x * 2 }

// 고차 함수 (함수를 매개변수로)
fun apply(value: Int, func: (Int) -> Int): Int {
    return func(value)
}

val result = apply(5) { it * 2 }  // 10

// 마지막 매개변수가 람다면 괄호 밖으로
val prices = listOf(50000, 48000, 52000)
val high = prices.filter { it > 49000 }

// 여러 줄 람다
val results = prices.map { price ->
    val tax = price * 0.1
    val fee = price * 0.01
    price - tax - fee  // 마지막 줄이 반환값
}
Tip: 중괄호 밖으로 빼기 (Trailing Lambda)

함수의 마지막 매개변수가 람다이면, 괄호 밖에 { }로 쓸 수 있습니다.
list.filter({ it > 0 }) 대신 list.filter { it > 0 }처럼 쓰면 훨씬 깔끔합니다.
이것이 Kotlin 코드가 DSL(도메인 특화 언어)처럼 읽히는 비결입니다.

11 companion object vs @classmethod / @staticmethod

Python
class Order:
    # 클래스 변수
    MAX_LEVERAGE = 100

    def __init__(self, symbol: str, size: int):
        self.symbol = symbol
        self.size = size

    # 팩토리 메서드 (classmethod)
    @classmethod
    def from_dict(cls, data: dict) -> "Order":
        return cls(data["symbol"], data["size"])

    # 유틸 메서드 (staticmethod)
    @staticmethod
    def validate_symbol(symbol: str) -> bool:
        return "_" in symbol

# 사용
order = Order.from_dict({"symbol": "BTC_USDT", "size": 1})
valid = Order.validate_symbol("BTC_USDT")
print(Order.MAX_LEVERAGE)  # 100
Kotlin
class Order(
    val symbol: String,
    val size: Int
) {
    // companion object = 클래스에 딸린 싱글톤
    companion object {
        // 클래스 상수
        const val MAX_LEVERAGE = 100

        // 팩토리 메서드
        fun fromMap(data: Map<String, Any>): Order {
            return Order(
                data["symbol"] as String,
                data["size"] as Int
            )
        }

        // 유틸 메서드
        fun validateSymbol(symbol: String): Boolean {
            return "_" in symbol
        }
    }
}

// 사용 (Python과 동일한 느낌!)
val order = Order.fromMap(mapOf("symbol" to "BTC_USDT", "size" to 1))
val valid = Order.validateSymbol("BTC_USDT")
println(Order.MAX_LEVERAGE)  // 100
핵심 요약

companion object = Python의 @classmethod + @staticmethod + 클래스 변수를 모아놓은 곳.
클래스 안에 하나만 존재할 수 있고, ClassName.method() 형태로 호출합니다.

12 코루틴 기초

Python의 asyncio를 써봤다면 Kotlin 코루틴도 빠르게 이해할 수 있습니다. 이 프로젝트에서는 kotlinx-coroutines-core 라이브러리를 사용합니다.

Python: asyncio
import asyncio

# async 함수 선언
async def fetch_price(symbol: str) -> float:
    await asyncio.sleep(1)  # 네트워크 대기 시뮬
    return 50000.0

# async 함수 안에서만 await 가능
async def main():
    price = await fetch_price("BTC_USDT")
    print(f"가격: {price}")

# 실행
asyncio.run(main())

# 동시 실행 (병렬)
async def main_parallel():
    prices = await asyncio.gather(
        fetch_price("BTC_USDT"),
        fetch_price("ETH_USDT"),
    )
    print(prices)
Kotlin: Coroutines
import kotlinx.coroutines.*

// suspend 함수 = async def
suspend fun fetchPrice(symbol: String): Double {
    delay(1000)  // 네트워크 대기 시뮬
    return 50000.0
}

// suspend 함수 안에서만 다른 suspend 호출 가능
suspend fun main() {
    val price = fetchPrice("BTC_USDT")
    println("가격: $price")
}

// 동시 실행 (병렬)
suspend fun mainParallel() = coroutineScope {
    val btc = async { fetchPrice("BTC_USDT") }
    val eth = async { fetchPrice("ETH_USDT") }

    println("BTC: ${btc.await()}")
    println("ETH: ${eth.await()}")
}

핵심 요약

suspend = Python의 async. 이 함수는 "중단될 수 있다"는 표시입니다.
Python과 달리 await 키워드가 필요 없습니다 — suspend 함수를 호출하면 자동으로 대기합니다.
병렬 실행은 async { }로 시작하고 .await()로 결과를 받습니다.

Tip: 이 프로젝트에서 코루틴을 쓰는 이유

gate-futures-bot은 거래소 API를 주기적으로 호출합니다. 네트워크 IO를 기다리는 동안 다른 작업(다른 계약의 가격 체크 등)을 할 수 있게 코루틴을 사용합니다. Python에서 aiohttp로 비동기 HTTP를 쓰는 것과 같은 맥락입니다.
