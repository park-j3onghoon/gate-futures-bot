1 이 단계의 목표

Gate.io 선물 계약에 롱(매수)과 숏(매도)을 주문할 수 있게 만듭니다. 포지션이 없을 때만 진입하고, 레버리지는 주문 직전에 설정합니다.

용어
  • 롱(Long): 가격이 오를 것에 베팅. size > 0
  • 숏(Short): 가격이 내릴 것에 베팅. size < 0
  • 레버리지: 증거금의 배수만큼 포지션을 잡는 배율
  • IOC: Immediate-or-Cancel. 즉시 체결 못 하면 취소

2 SDK 격리: 왜 래퍼가 필요한가

Gate.io는 Java SDK(io.gate:gate-api:7.1.8)를 제공합니다. 직접 쓰면 편하지만, 프로젝트 전체가 SDK 타입(FuturesOrder, GateApiException)에 묶입니다. SDK 버전을 올리거나 다른 거래소로 교체할 때 전체를 수정해야 합니다.

해결: 모든 SDK 호출을 GateClient에만 가둡니다. 다른 레이어(FuturesTrader, Strategy 등)는 우리 도메인 모델(OrderResult, Position)만 보게 합니다.

3 GateClient 구현

핵심 메서드 3가지:

  • createOrder(contract, size, price, tif): 주문 생성. size 양수=롱, 음수=숏
  • getPosition(contract): 현재 포지션 조회. size=0이면 null
  • updateLeverage(contract, leverage): 레버리지 설정
@Component
class GateClient(
    private val apiProperties: ApiProperties,
    internal val futuresApi: FuturesApi
) {
    private val logger = LoggerFactory.getLogger(GateClient::class.java)

    companion object {
        const val MARKET_PRICE = "0"
    }

    fun createOrder(
        contract: String,
        size: Long,
        price: String = MARKET_PRICE,
        tif: FuturesOrder.TifEnum = FuturesOrder.TifEnum.IOC
    ): OrderResult {
        return withRetry("createOrder(contract=$contract, size=$size)") {
            try {
                val order = FuturesOrder().apply {
                    this.contract = contract
                    this.size = size
                    this.price = price
                    this.tif = tif
                }
                val result = futuresApi.createFuturesOrder(apiProperties.settle, order, null)
                OrderResult.from(result)
            } catch (e: GateApiException) {
                throw mapGateException(e, "createOrder(contract=$contract, size=$size)")
            }
        }
    }
}
MARKET_PRICE = "0"

Gate.io API에서 price: "0"은 "시장가"를 의미합니다. 시장가 + IOC로 즉시 체결을 노립니다.

RateLimit 재시도

Gate.io는 API 호출 빈도가 높으면 RATE_LIMIT 에러를 반환합니다. 1회만 재시도:

private fun <T> withRetry(context: String, action: () -> T): T {
    return try {
        action()
    } catch (e: RateLimitException) {
        logger.warn("Rate limit 도달, 1.5초 후 재시도: {}", context)
        Thread.sleep(1500)
        action()
    }
}

4 sealed class 예외 계층

Gate.io가 반환하는 에러 라벨(errorLabel)을 도메인 예외로 매핑합니다. sealed class는 Kotlin의 enum 강화판으로, 허용된 하위 타입만 만들 수 있습니다. 

sealed class GateFuturesException(message: String, cause: Throwable? = null)
    : RuntimeException(message, cause)

class AuthenticationException(message: String, cause: Throwable? = null)
    : GateFuturesException(message, cause)

class InsufficientBalanceException(message: String, cause: Throwable? = null)
    : GateFuturesException(message, cause)

class OrderException(message: String, cause: Throwable? = null)
    : GateFuturesException(message, cause)

class RateLimitException(message: String, cause: Throwable? = null)
    : GateFuturesException(message, cause)

class PositionException(message: String, cause: Throwable? = null)
    : GateFuturesException(message, cause)

Gate.io 에러 라벨 → 도메인 예외 매핑:

private fun mapGateException(e: GateApiException, context: String): GateFuturesException {
    val label = e.errorLabel ?: ""
    val message = e.errorMessage ?: "Unknown Gate.io API error"
    return when (label) {
        "INVALID_KEY" -> AuthenticationException("[$context] $message", e)
        "BALANCE_NOT_ENOUGH" -> InsufficientBalanceException("[$context] $message", e)
        "RATE_LIMIT" -> RateLimitException("[$context] $message", e)
        else -> OrderException("[$context] $message", e)
    }
}
Python 비교

Python의 class MyException(Exception): pass와 유사. 차이는 sealed class는 같은 파일 내(또는 모듈 내)에서만 상속 가능 — 외부에서 새 하위 타입을 추가할 수 없어 when 분기의 exhaustiveness를 보장합니다.

5 FuturesTrader: 롱/숏 진입

비즈니스 레이어. GateClient를 호출하되, "이미 포지션 있으면 진입 거부", "레버리지 먼저 설정" 같은 규칙을 적용합니다. 

@Service
class FuturesTrader(private val client: GateClient) {

    fun openLong(contract: String, size: Int, leverage: Int = 5): OrderResult =
        openPosition(contract, size.toLong(), leverage, "롱")

    fun openShort(contract: String, size: Int, leverage: Int = 5): OrderResult =
        openPosition(contract, -size.toLong(), leverage, "숏")

    private fun openPosition(
        contract: String, size: Long, leverage: Int, direction: String
    ): OrderResult {
        val currentPosition = client.getPosition(contract)
        if (currentPosition != null) {
            throw PositionException("이미 포지션이 존재합니다: contract=$contract")
        }
        client.updateLeverage(contract, leverage)
        return client.createOrder(contract, size)
    }
}
롱/숏 한 함수로 처리

숏은 음수 size로 표현됩니다 (-size.toLong()). Gate.io가 size 부호로 방향을 결정하므로 한 함수로 통합 가능.

6 MockK 테스트 패턴

Kotlin 전용 목킹 라이브러리 MockK로 FuturesApi를 가짜로 만듭니다. FuturesApi는 Java 클래스이지만 MockK가 Java/Kotlin 모두 지원. 

class FuturesTraderTest {
    private lateinit var trader: FuturesTrader
    private lateinit var client: GateClient

    @BeforeEach
    fun setUp() {
        client = mockk()
        trader = FuturesTrader(client)
    }

    @Test
    fun `should open long position successfully`() {
        every { client.getPosition("BTC_USDT") } returns null
        every { client.updateLeverage("BTC_USDT", 5) } returns Unit
        every { client.createOrder("BTC_USDT", 1L) } returns OrderResult(
            id = 1L, contract = "BTC_USDT", size = 1L, price = "0",
            status = "finished", fillPrice = "50000", createTime = 1700000000.0
        )

        val result = trader.openLong("BTC_USDT", 1, 5)

        assertEquals(1L, result.id)
        verify { client.createOrder("BTC_USDT", 1L) }
        verify { client.updateLeverage("BTC_USDT", 5) }
    }

    @Test
    fun `should throw PositionException when position already exists`() {
        val existing = Position("BTC_USDT", 1L, "50000", 5, "0", "0")
        every { client.getPosition("BTC_USDT") } returns existing

        assertThrows<PositionException> {
            trader.openLong("BTC_USDT", 1, 5)
        }
    }
}

MockK 핵심 DSL

패턴의미
mockk()인스턴스 생성
