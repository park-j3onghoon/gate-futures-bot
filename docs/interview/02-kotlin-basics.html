1 변수: val / var

Kotlin은 변수를 선언할 때 immutable(val)인지 mutable(var)인지 명시합니다. Python엔 없는 개념.

# Python: 모든 변수가 재할당 가능
name = "Teddy"
name = "Bob"  # OK

# 불변을 의도하려면 관례 또는 Final 힌트
from typing import Final
MAX: Final = 100
# 하지만 런타임엔 재할당 가능 (힌트일 뿐)
// Kotlin: val (read-only) vs var (mutable)
val name: String = "Teddy"
name = "Bob"  // 컴파일 에러!

var count: Int = 0
count = 1     // OK

// 타입 추론: 명시 생략 가능
val greeting = "Hello"  // String으로 자동 추론
var age = 30            // Int
팁: val 기본

실무에선 항상 val을 먼저 쓰고, 재할당이 필요할 때만 var로 바꿉니다. 대부분 변수는 불변으로 유지될 수 있으며, 이게 버그를 크게 줄여줍니다. IntelliJ가 "이 var는 val로 바꿀 수 있어"를 자동 제안합니다.

3 Null Safety (핵심 중의 핵심)

Kotlin의 가장 큰 가치. 타입 자체에 "null일 수 있는지" 표기합니다. Python/Java에서 "NoneType has no attribute" / "NullPointerException"은 가장 흔한 런타임 에러. Kotlin은 이걸 컴파일 타임에 차단합니다.

String
= 절대 null 아님
|
String?
= null일 수 있음
# Python: 타입 시스템이 강제 안함
def find_user(id: int) -> User | None:
    ...

user = find_user(1)
print(user.name)  # None이면 AttributeError (런타임)

# 방어 코드 필요
if user is not None:
    print(user.name)
fun findUser(id: Long): User? { /* ... */ }

val user = findUser(1)
println(user.name)  // 컴파일 에러!
// User? 타입이므로 null 체크 없이 .name 접근 불가

// 방법 1: safe call (?.) - null이면 null 반환
println(user?.name)

// 방법 2: elvis (?:) - null 대체값
println(user?.name ?: "익명")

// 방법 3: 명시적 체크 (smart cast)
if (user != null) {
    println(user.name)  // 이 블록 안에선 User로 취급
}

// 방법 4: !! (단언, 위험) - null이면 NPE
println(user!!.name)  // null이면 터짐

Null safety 연산자 총정리

?.
safe call. null이면 전체 표현식이 null
?:
elvis. 왼쪽이 null이면 오른쪽 사용
!!
단언. null이면 NPE. 사용 최소화
as?
safe cast. 실패시 null
let
x?.let { ... } — null이 아닐 때만 블록 실행
requireNotNull
null이면 IllegalArgumentException. 계약 위반용
!!는 최후의 수단

!!는 "내가 책임지고 null 아님을 보장한다"는 선언이지만, 실수하면 NPE로 직행합니다. 코드 리뷰에서 !!가 보이면 항상 "정말 필요한가?" 질문이 따라옵니다. 대개 ?: 또는 requireNotNull(...) { "명시적 에러 메시지" }로 대체합니다.

5 제어 흐름

if - 표현식

Kotlin에서 if는 값을 반환하는 표현식입니다 (Python 삼항 연산자에 가깝지만 더 강력).

score = 85
grade = "A" if score >= 90 else "B" if score >= 80 else "C"

# 문으로 쓰려면:
if score >= 90:
    print("A")
elif score >= 80:
    print("B")
else:
    print("C")
val score = 85

// if가 값 반환
val grade = if (score >= 90) "A"
            else if (score >= 80) "B"
            else "C"

// 블록도 가능 (마지막 줄이 값)
val msg = if (score >= 60) {
    println("통과 로그")
    "합격"
} else {
    "불합격"
}

when - 강력한 switch

Python match보다 먼저 나왔고 더 유연. exhaustive 검사가 핵심.

// 값 매칭
val day = 3
val name = when (day) {
    1 -> "월"
    2, 3 -> "화/수"
    in 4..5 -> "목/금"
    else -> "주말"
}

// 타입 매칭 (is) + 스마트 캐스트
fun describe(x: Any): String = when (x) {
    is Int -> "정수 ${x * 2}"      // x가 Int로 취급됨
    is String -> "문자열 길이 ${x.length}"
    is List<*> -> "리스트 ${x.size}개"
    else -> "알 수 없음"
}

// 조건식 (인자 없는 when)
val category = when {
    score >= 90 -> "우수"
    score >= 70 -> "양호"
    score < 0 -> throw IllegalArgumentException("음수")
    else -> "미달"
}

// sealed class와 exhaustive (else 불필요)
sealed class Result
object Success : Result()
data class Failure(val msg: String) : Result()

fun handle(r: Result) = when (r) {
    is Success -> "OK"
    is Failure -> "에러: ${r.msg}"
    // else 필요 없음: sealed라 모든 케이스 커버됨을 컴파일러가 확인
}

6 타입 체크와 스마트 캐스트

Kotlin의 별미. is로 체크하면 그 블록 안에서 자동으로 타입이 좁혀집니다 (Python엔 없는 기능).

fun process(value: Any) {
    if (value is String) {
        // 여기선 value가 String으로 자동 취급
        println(value.length)  // .length 접근 가능
    }

    // null 체크 + 스마트 캐스트
    val x: String? = getSomething()
    if (x != null) {
        println(x.length)  // String으로 좁혀짐, ?. 불필요
    }

    // 명시적 캐스트 (as) - 실패시 ClassCastException
    val s = value as String

    // safe cast - 실패시 null
    val s2 = value as? String  // String?
}

require / check / error

상황별로 쓰는 빌트인 함수들 — Python의 assert보다 의도가 명확합니다.

fun divide(a: Int, b: Int): Int {
    require(b != 0) { "b는 0이 아니어야 함" }      // IllegalArgumentException
    // 파라미터/인자 검증용

    val result = a / b
    check(result >= 0) { "결과가 음수" }            // IllegalStateException
    // 내부 상태 검증용

    if (result > 1_000_000) error("너무 큼")       // IllegalStateException
    // 도달 불가라 가정한 분기

    return result
}

// requireNotNull — null이면 IllegalArgumentException
val x = requireNotNull(maybeNull) { "x는 null이면 안됨" }
핵심 요약
  • val / var: 항상 val 먼저, 필요할 때만 var
  • 타입 필수: 함수 파라미터/리턴엔 타입 명시 (지역 변수는 추론 OK)
  • Null safety: T vs T? 타입으로 구분. ?., ?: 사용. !! 최소화
  • if/when은 표현식: 값을 반환하는 용도로 자주 씀
  • 스마트 캐스트: is / null 체크 후 자동 타입 좁힘
  • require/check/error: 의도를 명시한 방어 함수
