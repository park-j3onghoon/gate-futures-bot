1 캔들(OHLCV)이란

캔들스틱(Candlestick)은 일정 시간 동안의 가격 움직임을 하나의 막대로 요약합니다. 5개 값을 담습니다:

기호의미
O (Open)시작 가격
H (High)최고 가격
L (Low)최저 가격
C (Close)종료 가격
V (Volume)거래량
왜 종가(Close)만 쓰나

이 프로젝트의 지표(SMA/EMA/RSI)는 종가만 사용합니다. 다른 지표(볼린저 밴드, 스토캐스틱)는 고/저도 쓰지만 기본 전략엔 불필요.

2 Candle / Interval 모델

Gate.io SDK의 FuturesCandlestick은 필드가 null 가능(c: String?). 우리 도메인 모델에서는 필수 필드가 null이면 즉시 실패시켜 silent corruption을 방지합니다. 

data class Candle(
    val timestamp: Long,
    val open: String,
    val high: String,
    val low: String,
    val close: String,
    val volume: Long
) {
    val closePrice: Double get() = close.toDouble()

    companion object {
        fun from(c: FuturesCandlestick): Candle {
            val t = c.t ?: throw MarketDataException("timestamp(t) 없음")
            val close = c.c ?: throw MarketDataException("close(c) 없음")
            // ... o/h/l 동일
            return Candle(t.toLong(), open, high, low, close, c.v ?: 0L)
        }
    }
}

enum class Interval(val code: String) {
    SEC_10("10s"),
    MIN_1("1m"), MIN_5("5m"), MIN_15("15m"), MIN_30("30m"),
    HOUR_1("1h"), HOUR_4("4h"), HOUR_8("8h"),
    DAY_1("1d"), DAY_7("7d"), DAY_30("30d");
}
왜 가격을 String으로 저장하나

Gate.io API가 String으로 반환합니다 (정밀도 유지). Double 변환은 지표 계산 시점에만. 금액 비교는 String으로, 계산은 Double로 — 영역을 분리합니다.

3 GateClient.getCandlesticks

Gate.io API는 interval은 필수, limit/from/to는 선택. Kotlin ?.let로 optional 체이닝: 

fun getCandlesticks(
    contract: String,
    interval: Interval,
    limit: Int? = null,
    fromSec: Long? = null,
    toSec: Long? = null
): List<Candle> {
    return try {
        val request = futuresApi.listFuturesCandlesticks(apiProperties.settle, contract)
            .interval(interval.code)
        limit?.let { request.limit(it) }
        fromSec?.let { request.from(it) }
        toSec?.let { request.to(it) }

        val candles = request.execute()
        candles.map { Candle.from(it) }
    } catch (e: GateApiException) {
        throw mapGateException(e, "getCandlesticks(contract=$contract, interval=${interval.code})")
    }
}

4 SMA / EMA / RSI 공식과 구현

Top-level 함수로 작성합니다. Spring 의존 없는 순수 함수 → 어디서나 호출 가능하고 테스트도 쉬움.

SMA (Simple Moving Average)

SMA = (p₁ + p₂ + ... + pₙ) / n — 최근 n개 가격의 평균.

fun calculateSma(prices: List<Double>, period: Int): Double? {
    require(period > 0) { "period must be positive" }
    if (prices.size < period) return null
    return prices.takeLast(period).average()
}

EMA (Exponential Moving Average)

최근 값에 더 큰 가중치. 계산:
multiplier = 2 / (period + 1)
EMA = (current - prev_ema) × multiplier + prev_ema 

fun calculateEma(prices: List<Double>, period: Int): Double? {
    require(period > 0)
    if (prices.size < period) return null

    val multiplier = 2.0 / (period + 1)
    var ema = prices.take(period).average() // seed: 첫 period의 SMA
    for (i in period until prices.size) {
        ema = (prices[i] - ema) * multiplier + ema
    }
    return ema
}

RSI (Relative Strength Index)

0~100 범위. 70+ = 과매수(곧 떨어질 가능성), 30- = 과매도(곧 오를 가능성).
Wilder's smoothing 공식:

  • 첫 period: gain/loss 단순 평균
  • 이후: avg = (prev_avg × (period-1) + current) / period
  • RS = avgGain / avgLoss, RSI = 100 - 100/(1 + RS)
fun calculateRsi(prices: List<Double>, period: Int = 14): Double? {
    require(period > 0)
    if (prices.size <= period) return null

    var avgGain = 0.0
    var avgLoss = 0.0
    for (i in 1..period) {
        val change = prices[i] - prices[i - 1]
        if (change >= 0) avgGain += change else avgLoss -= change
    }
    avgGain /= period
    avgLoss /= period

    for (i in period + 1 until prices.size) {
        val change = prices[i] - prices[i - 1]
        avgGain = (avgGain * (period - 1) + max(change, 0.0)) / period
        avgLoss = (avgLoss * (period - 1) + max(-change, 0.0)) / period
    }

    // float equality 대신 epsilon 비교
    val lossIsZero = abs(avgLoss) < 1e-12
    val gainIsZero = abs(avgGain) < 1e-12
    if (lossIsZero) return if (gainIsZero) 50.0 else 100.0
    val rs = avgGain / avgLoss
    return 100.0 - (100.0 / (1.0 + rs))
}
부동소수점 함정

avgLoss == 0.0처럼 Double을 직접 비교하면 IEEE 754 누적 오차로 1e-16 같은 미세값을 놓칠 수 있습니다. abs(x) < 1e-12 형태의 epsilon 비교를 씁니다.

5 MarketDataService

Spring @Service. GateClient를 호출하여 캔들과 최신 가격을 제공. 빈 응답은 예외로: 

@Service
class MarketDataService(private val client: GateClient) {

    fun getCandles(
        contract: String, interval: Interval,
        limit: Int? = null, fromSec: Long? = null, toSec: Long? = null
    ): List<Candle> = client.getCandlesticks(contract, interval, limit, fromSec, toSec)

    fun getLatestPrice(contract: String, interval: Interval = Interval.MIN_1): Double {
        val candles = client.getCandlesticks(contract, interval, limit = 1)
        val latest = candles.lastOrNull()
            ?: throw MarketDataException("캔들 없음: $contract")
        return latest.closePrice
    }
}

@Test
fun `RSI equals 100 when all prices strictly increase`() {
    val prices = (1..20).map { it.toDouble() }
    assertEquals(100.0, calculateRsi(prices, period = 14))
}

@Test
fun `RSI returns 50 when no price changes`() {
    val prices = List(20) { 100.0 }
    assertEquals(50.0, calculateRsi(prices, period = 14))
}
