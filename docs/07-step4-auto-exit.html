1 왜 ExitCondition은 별도 타입인가

진입용 EntryCondition과 청산용 ExitCondition은 필드가 비슷하더라도 의미적으로 다른 타입입니다. 같은 data class로 묶으면 실수로 진입 조건을 청산에 넣을 수 있습니다.

타입으로 강제하기

TradingStrategy(longEntries: List<EntryCondition>, exitConditions: List<ExitCondition>)처럼 파라미터 타입을 다르게 하면 컴파일 타임에 혼용 방지. Kotlin의 타입 시스템을 버그 예방에 활용합니다.

2 PnL 공식: 롱과 숏의 대칭

PnL(Profit and Loss)은 명목가(nominal) 기준 수익률입니다. 레버리지는 증거금 배율이지, PnL% 자체는 가격 변동만 반영합니다.

포지션PnL 공식예: entry=100, current=105
롱 (size > 0) (current - entry) / entry × 100 (105-100)/100×100 = +5%
숏 (size < 0) (entry - current) / entry × 100 (100-105)/100×100 = -5% 
internal fun pnlPercent(position: Position, currentPrice: Double): Double {
    val entry = position.entryPrice.toDoubleOrNull()
    if (entry == null || entry == 0.0) {
        logger.warn("비정상 entryPrice: ${position.entryPrice}")
        return 0.0
    }
    return if (position.size > 0) {
        (currentPrice - entry) / entry * 100.0
    } else {
        (entry - currentPrice) / entry * 100.0  // 부호 반대
    }
}
Inverse 계약은 다르다

위 공식은 Linear USDT-margined 계약(BTC_USDT)용입니다. Inverse 계약(BTC_USD)은 (1/entry - 1/current) × size로 계산이 다릅니다. 이 프로젝트는 Linear만 지원.

3 ExitCondition sealed 계층

3가지 서브 타입. 각자 evaluate를 구현:

sealed class ExitCondition {
    abstract fun evaluate(prices: List<Double>, position: Position): Boolean

    data class TakeProfitPct(val pct: Double) : ExitCondition() {
        override fun evaluate(prices: List<Double>, position: Position): Boolean {
            val current = prices.lastOrNull() ?: return false
            return pnlPercent(position, current) >= pct
        }
    }

    data class StopLossPct(val pct: Double) : ExitCondition() {
        override fun evaluate(prices: List<Double>, position: Position): Boolean {
            val current = prices.lastOrNull() ?: return false
            return pnlPercent(position, current) <= -pct  // pct는 양수로 전달
        }
    }

    data class IndicatorExit(
        val indicator: Indicator,
        val operator: ComparisonOp,
        val value: Double,
        val period: Int = 14
    ) : ExitCondition() {
        // EntryCondition과 동일 로직이지만 의미는 청산
        override fun evaluate(prices: List<Double>, position: Position): Boolean {
            // ... RSI/SMA/EMA/PRICE 계산 후 비교
        }
    }
}

4 evaluateExit: OR 결합

Entry는 AND(모든 조건 충족 시 진입), Exit는 OR(하나라도 충족 시 청산).
이유: 청산은 손실 제한 목적. 한 가지 위험 신호만 있어도 빠져나가는 게 안전. 

fun evaluateExit(candles: List<Candle>, position: Position): ExitSignal {
    if (candles.isEmpty() || exitConditions.isEmpty()) return ExitSignal.None
    val prices = candles.map { it.closePrice }
    val triggered = exitConditions.filter { it.evaluate(prices, position) }
    return if (triggered.isNotEmpty()) ExitSignal.Close(triggered) else ExitSignal.None
}

sealed class ExitSignal {
    data object None : ExitSignal()
    data class Close(val triggered: List<ExitCondition>) : ExitSignal()
}

5 GateClient.closePosition: close=true

Gate.io 선물은 FuturesOrder.close = true와 size = 0으로 전달하면 현재 포지션 전체를 시장가로 청산합니다. 방향은 서버가 자동 판정. 

fun closePosition(contract: String): OrderResult {
    return withRetry("closePosition(contract=$contract)") {
        try {
            val order = FuturesOrder().apply {
                this.contract = contract
                this.size = 0L
                this.price = MARKET_PRICE
                this.tif = FuturesOrder.TifEnum.IOC
                this.close = true
            }
            val result = futuresApi.createFuturesOrder(apiProperties.settle, order, null)
            OrderResult.from(result)
        } catch (e: GateApiException) {
            throw mapGateException(e, "closePosition(contract=$contract)")
        }
    }
}

6 CoinWorker 전체 사이클

한 iteration이 포지션 유무에 따라 분기:

internal suspend fun runOnce() {
    try {
        rateLimiter.acquire()
        val position = trader.getCurrentPosition(contract)
        updateCandles()
        if (position != null) {
            evaluateAndClose(position)  // 청산 평가
        } else {
            evaluateAndEnter()          // 진입 평가
        }
    } catch (e: CancellationException) {
        throw e  // 코루틴 취소는 그대로 전파
    } catch (e: Exception) {
        logger.error("워커 iteration 에러: contract=$contract", e)
    }
}
