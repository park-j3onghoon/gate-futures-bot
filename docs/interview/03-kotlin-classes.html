1 일반 class와 생성자

Kotlin 클래스는 Java보다 훨씬 간결합니다. primary constructor가 클래스 헤더에 바로 선언됩니다.

class User:
    def __init__(self, id: int, name: str, email: str = ""):
        self.id = id
        self.name = name
        self.email = email
        self._cache = {}

    def __repr__(self):
        return f"User({self.id}, {self.name})"

    def greet(self) -> str:
        return f"Hello, {self.name}"

u = User(1, "Teddy")
class User(
    val id: Long,
    val name: String,
    val email: String = ""  // default 값
) {
    private val cache = mutableMapOf<String, Any>()

    // init 블록: primary constructor 직후 실행
    init {
        require(name.isNotBlank()) { "name required" }
    }

    fun greet(): String = "Hello, $name"
}

val u = User(1, "Teddy")  // new 키워드 없음
val u2 = User(id = 1, name = "Teddy")  // named args
Primary Constructor의 파라미터가 곧 프로퍼티

헤더에 val/var을 붙이면 자동으로 프로퍼티가 됩니다. Python에서 self.x = x를 일일이 쓰는 것보다 훨씬 깔끔. val을 빼면 생성자 인자일 뿐(외부 접근 불가).

Secondary constructor (여러 생성자)

class User(val id: Long, val name: String) {
    // secondary: primary를 this(...)로 위임 호출 필수
    constructor(name: String) : this(id = 0, name = name)
    constructor() : this("anonymous")
}

프로퍼티 getter/setter 커스텀

class Rectangle(val width: Int, val height: Int) {
    // computed property (getter)
    val area: Int
        get() = width * height

    // backing field로 setter 커스텀
    var description: String = ""
        get() = field.uppercase()
        set(value) {
            field = value.trim()
        }
}

2 data class

Python의 @dataclass에 해당. equals/hashCode/toString/copy를 자동 생성. DTO, VO, 응답 객체에 압도적으로 쓰임. 

from dataclasses import dataclass, field

@dataclass(frozen=True)
class Money:
    amount: int
    currency: str = "KRW"

m1 = Money(1000)
m2 = Money(1000)
m1 == m2  # True (dataclass가 __eq__ 생성)

# 복사 + 수정
import dataclasses
m3 = dataclasses.replace(m1, amount=2000)
data class Money(
    val amount: Long,
    val currency: String = "KRW"
)

val m1 = Money(1000)
val m2 = Money(1000)
m1 == m2  // true (equals 자동)
println(m1)  // "Money(amount=1000, currency=KRW)"

// 복사 + 수정
val m3 = m1.copy(amount = 2000)

// 구조 분해 (destructuring)
val (amount, currency) = m1

data class 자동 생성 메서드

  • equals(other): 모든 프로퍼티 비교
  • hashCode(): 해시 기반 컬렉션용
  • toString(): 디버깅용 표현
  • copy(...): 일부만 변경한 새 인스턴스
  • componentN(): 구조 분해(val (a, b) = data)
언제 data class가 아닌가

data class는 "값 객체"용. 아래 경우엔 일반 class 사용:

  • JPA Entity: Hibernate가 data class의 equals를 혼란스러워 함 (id만 비교하는 게 일반적)
  • 상속 계층이 필요한 경우: data class는 open 불가 (Kotlin 1.1+에서 제약 완화됐지만 관행상 피함)
  • 동작이 핵심인 Service 클래스

3 상속과 interface

Kotlin 클래스는 기본 final(상속 불가). 상속을 허용하려면 open 수식어 필수. Java/Python(모두 open 기본)과 반대. 

class Animal:
    def __init__(self, name):
        self.name = name
    def sound(self) -> str:
        return "..."

class Dog(Animal):
    def sound(self) -> str:
        return "멍멍"

# Interface는 ABC나 Protocol
from abc import ABC, abstractmethod
class Greeter(ABC):
    @abstractmethod
    def greet(self) -> str: ...
// 상속 가능하게 하려면 open 필수
open class Animal(val name: String) {
    open fun sound(): String = "..."
}

class Dog(name: String) : Animal(name) {
    override fun sound(): String = "멍멍"
}

// interface
interface Greeter {
    fun greet(): String
    // default 구현 가능
    fun hi() = "Hi, ${greet()}"
}

class EnglishGreeter : Greeter {
    override fun greet() = "Hello"
}

abstract class

abstract class Shape {
    abstract fun area(): Double  // 반드시 구현 필요
    open fun describe(): String = "도형"  // override 선택
}

class Circle(val radius: Double) : Shape() {
    override fun area() = Math.PI * radius * radius
}

다중 interface 구현

interface Readable {
    fun read(): String
}
interface Writable {
    fun write(s: String)
}

class File : Readable, Writable {
    override fun read() = "..."
    override fun write(s: String) { /* ... */ }
}

4 sealed class (⭐ 핵심)

"닫힌 클래스 계층". 허용된 하위 타입만 존재하고, 같은 모듈(또는 같은 파일)에서만 상속 가능. Python의 enum을 확장한 개념 + 대수적 데이터 타입.

실전 사용처: 상태 머신, Result 타입, 이벤트, 도메인 signal.

// sealed class로 결과 타입 정의
sealed class ApiResult<out T> {
    data class Success<T>(val data: T) : ApiResult<T>()
    data class Failure(val code: Int, val message: String) : ApiResult<Nothing>()
    object Loading : ApiResult<Nothing>()
}

// when이 exhaustive: 모든 케이스 커버 필수 (else 불필요)
fun handle(result: ApiResult<User>): String = when (result) {
    is ApiResult.Success -> "데이터: ${result.data}"
    is ApiResult.Failure -> "에러 ${result.code}: ${result.message}"
    ApiResult.Loading -> "로딩중"
    // 새 하위 타입 추가하면 컴파일 에러로 이곳을 수정 강제
}
sealed class ApiResult<T>
Success<T>(data)
Failure(code, msg)
Loading
왜 enum이 아니고 sealed?

enum은 값의 집합(각 상수가 같은 구조). sealed는 타입의 집합(각 하위 타입이 다른 필드 보유 가능). 예: Failure는 code/message를 가지지만 Loading은 아무것도 안 가짐 → sealed가 맞음.

5 object / companion object

object (싱글턴)

Python의 모듈 레벨 싱글턴을 "언어 기본 기능"으로 제공.

# 관용적 싱글턴 (모듈 = 싱글턴)
class _Config:
    host = "localhost"
    port = 8080

config = _Config()  # 모듈 로드시 1회

# 사용
from myapp import config
print(config.host)
// object 선언 = 싱글턴 자동
object Config {
    val host: String = "localhost"
    val port: Int = 8080

    fun describe() = "$host:$port"
}

// 사용
println(Config.host)
println(Config.describe())

companion object (클래스의 static 영역)

Java의 static이 Kotlin엔 없고, 클래스 안의 companion object로 대체.

class User(val id: Long, val name: String) {
    companion object {
        const val MAX_NAME_LENGTH = 30

        // 팩토리 메서드
        fun from(raw: Map<String, Any>): User = User(
            id = raw["id"] as Long,
            name = raw["name"] as String
        )
    }
}

object expression (익명 객체)

// Java 익명 클래스에 해당. 1회성 구현
val comparator = object : Comparator<User> {
    override fun compare(a: User, b: User) = a.id.compareTo(b.id)

// 단일 함수 인터페이스(SAM)면 람다로 더 간단

6 enum class

from enum import Enum

class Status(Enum):
    ACTIVE = "active"
    INACTIVE = "inactive"
    PENDING = "pending"

Status.ACTIVE.value  # "active"

# 메서드 있는 enum
class Direction(Enum):
    NORTH = (0, 1)
    SOUTH = (0, -1)

    def dx(self): return self.value[0]
    def dy(self): return self.value[1]
enum class Status(val code: String) {
    ACTIVE("active"),
    INACTIVE("inactive"),
    PENDING("pending");

    fun isUsable() = this != INACTIVE
}

Status.ACTIVE.code          // "active"
Status.ACTIVE.isUsable()    // true
Status.values()             // 모든 값
Status.valueOf("ACTIVE")    // 이름으로 조회

// 추상 메서드 있는 enum
enum class Direction(val dx: Int, val dy: Int) {
    NORTH(0, 1), SOUTH(0, -1), EAST(1, 0), WEST(-1, 0);
}

7 가시성 수식어

Python은 관례(밑줄)로 비공개 표기, Kotlin은 컴파일러가 강제.

수식어Python 대응범위
public (기본) 일반 이름 어디서나 접근
internal 해당 없음 같은 모듈(Gradle 모듈) 내에서만
protected _name (관례) 해당 클래스 + 하위 클래스
private __name (관례) 해당 클래스(또는 파일) 내부만
실전 정리
  • 일반 class: 상태 + 동작. JPA Entity, Service 등 (기본 final, open 필요시 명시)
  • data class: 불변 값 객체(DTO, 응답, 설정 스펙)
  • sealed class: 닫힌 계층(결과 타입, 상태, 이벤트) + exhaustive when
  • object: 싱글턴. 유틸, 설정, 전역 state 보유
  • companion object: 클래스 상수 + 팩토리 메서드. Java static 대체
  • enum class: 유한한 값 집합. 메서드/프로퍼티 보유 가능
  • interface: 다중 구현 가능, default 메서드 허용
